# ADR-0007 – Single Codebase: src/

Status: Accepted

## Context

The repository contained two parallel Python packages at the root level:

- `app/` – the real SQLAlchemy implementation (models, core, repositories, services, CLI).
- `src/` – a domain-module skeleton (analytics, crm, finance, etc.) intended as the long-term structure.

This split caused three concrete problems:

1. Code coverage was configured as `--cov=src` but all real code lived in `app/`, so
   coverage always measured empty stubs.
2. Tests imported from `app.*`, making it impossible to measure them.
3. Future contributors had to understand two parallel hierarchies.

## Decision

All code is consolidated in `src/`.

`src/` uses a hybrid layout:

- Infrastructure lives in `src/core/`, `src/repositories/`, `src/services/`, `src/cli/`.
- Persistent domain models live in `src/models/` (flat, matching DOMAIN_DICTIONARY).
- Future domain modules (`src/crm/`, `src/products/`, etc.) contain domain logic and
  will reference `src/models/` for persistence.

The `app/` directory is removed entirely.

## Consequences

- `--cov=src` now measures real production code.
- All imports use the `src.*` namespace.
- The `src/models/` flat layout is considered temporary; a future ADR may distribute
  models into their domain modules once the domain layer matures.
