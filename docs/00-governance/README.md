# 00-governance – Styrande dokument

## Syfte

Governance-dokumenten definierar **varför** AlpenTind Platform byggs och **hur** projektet styrs. De ligger i toppen av dokumenthierarkin och vägleder alla beslut nedåt.

---

## Varför governance?

Utan tydliga grundprinciper riskerar ett projekt att:

- Fatta beslut som strider mot affärsmålen
- Bygga tekniska lösningar utan förankring i verksamheten
- Tappa sammanhang när teamet förändras

Governance-dokumenten ger projektet ett stabilt fundament.

---

## Varför ligger de över övriga dokument?

Governance definierar spelreglerna. Arkitektur, design och kod måste följa dessa regler – inte tvärtom.

Se [DOKUMENTHIERARKI.md](./DOKUMENTHIERARKI.md) för hela kedjan.

## Regel

Alla dokument i lägre nivåer (business, architecture, design, kod) måste följa de principer som definieras här.

Om ett lägre dokument verkar strida mot ett governance-dokument gäller governance-dokumentet.
