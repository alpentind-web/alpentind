# Workspace Architecture

**Version:** 1.0  
**Status:** Approved

## Purpose

A Workspace is the primary operational environment inside AlpenTind Platform.

It is not a page or a form.

Every Workspace answers four operational questions.

## Four Operational Questions

1. What is the current situation? → Health
2. What needs attention? → Work Queue
3. What do I need to know? → Context
4. What should I do next? → Actions

## Operational Blocks

- Health
- Work Queue
- Timeline
- Context
- Actions
- Notes

## Workspace Principles

- Situation before interaction
- Work before administration
- Calm before complexity
- Everything has context
- Actions follow understanding
- Consistency over novelty
- Every Workspace starts with a question

## Current Workspaces

- Arbetsdag
- Dialog
- Experience (planned)
- Person (planned)
- Guide (planned)

## Design Goal

Every Workspace should feel calm, reduce uncertainty and make the next action obvious.
